Приклад: author --Надає посилання на автора документа dns-prefetch --Вказує, що браузер повинен попередньо виконувати роздільну здатність DNS для джерела цільового ресурсу help --Надає посилання на довідковий документ. Приклад: icon --мпортує значок для представлення документа. Приклад: license --Надає посилання на інформацію про авторські права на документ next --Надає посилання на наступний документ із серії pingback --Надає адресу сервера pingback, який обробляє pingbacks поточному документу preconnect --Вказує, що браузер повинен попередньо підключитися до джерела цільового ресурсу. prefetch --Вказує, що веб-переглядач повинен попередньо вибирати та кешувати цільовий ресурс, оскільки це, ймовірно, буде потрібно для подальшої навігації preload --Вказує, що агент браузера повинен попередньо отримати та кешувати цільовий ресурс для поточної навігації відповідно до пункту призначення, заданого атрибутом "as" (та пріоритету, пов'язаного з цим призначенням). prerender --Вказує, що браузер повинен попередньо відобразити (завантажити) вказану веб-сторінку у фоновому режимі. Отже, якщо користувач переходить на цю сторінку, це прискорює завантаження сторінки (оскільки сторінка вже завантажена). Увага! Це втрачає пропускну здатність користувача! Використовуйте попередній візуалізацію лише в тому випадку, якщо ви абсолютно впевнені, що веб-сторінка потрібна в якийсь момент шляху користувача prev --Вказує, що документ є частиною серії і що попередній документ у серії є посилальним документом search --Надає посилання на ресурс, який може бути використаний для пошуку через поточний документ та пов’язані з ним сторінки. ----------icon HeightxWidth --изначає один або більше розмірів для пов'язаної піктограми. Значення висоти та ширини розділені знаками "x" або "X". Приклади: (1 size) (2 sizes) any --Вказує, що піктограма може бути масштабованою (як SVG-зображення) Приклади: (any size) --------------------------------media all - За замовчуванням. Використовується для всіх пристроїв мультимедійного типу speech - Використовується для читачів екрана, які "читають" сторінку вголос print - Використовується для режиму попереднього перегляду друку / друкованих сторінок screen - Використовується для екранів комп'ютерів, планшетів, смартфонів тощо. ---------screan aspect-ratio --Визначає співвідношення ширини / висоти цільової області відображення. Префікси "min-" та "max-" можна використовувати. Приклад: media="screen and (max-aspect-ratio:16/9)" color --Вказує біти на колір відображення цілі. Префікси "min-" та "max-" можна використовувати. Приклад: media="screen and (min-color:3)" color-index -- Вказує кількість кольорів, з якими може працювати цільовий дисплей. Префікси "min-" та "max-" можна використовувати. Приклад: media="screen and (min-color-index:256)" grid -- Вказує, чи пристроєм виведення є сітка чи растрова карта. Можливі значення "1" для сітки, а "0" в іншому випадку. Приклад: media="handheld and (grid:1)" height --Вказує висоту цільової області відображення. Префікси "min-" та "max-" можна використовувати. Приклад: media="screen and (max-height:700px)" monochrome --Визначає біти на піксель у монохромному буфері кадру. Префікси "min-" та "max-" можна використовувати. Приклад: media="screen and (min-monochrome:2)" orientation --Визначає орієнтацію цільового дисплея / паперу. Можливі значення: "портрет" або "пейзаж" ("portrait" or "landscape") Приклад: media="all and (orientation: landscape)" resolution --Визначає щільність пікселів (dpi або dpcm) цільового дисплея / паперу. Префікси "min-" та "max-" можна використовувати. Приклад: media="print and (min-resolution:300dpi)" scan --Вказує спосіб сканування телевізійного дисплея. Можливі значення "прогресивне" та "переплетення"("progressive" and "interlace"). Приклад: media="tv and (scan:interlace)" width --Вказує ширину цільової області відображення. Префікси "min-" та "max-" можна використовувати.
